1. But et Objectif du Projet

Ce projet est une "Banque de Patrons" (Design Patterns). Il ne sert pas juste à créer des drones, il sert à montrer comment Java fonctionne sur les sujets qui tombent TOUJOURS à l'examen.

L'objectif : Que tu puisses trouver en moins de 30 secondes le code correspondant à une question du type : "Écrivez une méthode qui calcule la moyenne de X", ou "Créez une exception personnalisée".

2. Présentation du Projet

Le projet simule une entreprise qui gère des drones. Il y a :

  • Le Contrat (Interface) : Ce que tous les drones doivent faire.
  • Le Modèle (Héritage) : Une base commune pour ne pas répéter le code.
  • La Gestion (Collections) : Le cerveau qui stocke, cherche et trie.
  • La Sécurité (Exceptions) : Ce qui se passe quand quelque chose va mal.

3. Explication Globale

Tout est centralisé dans Flotte.java. C'est là que se trouvent les algorithmes. Les autres fichiers servent à définir la structure des données (les "objets").
